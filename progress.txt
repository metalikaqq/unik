# Ralph Progress Log

## Codebase Patterns

- **Tailwind v4 CSS-first (ADR-0005)**: Tokens live in `src/app/globals.css`.
  Pattern is `:root { --color-bg: #FAFAF7; ... }` + `@theme inline { --color-bg: var(--color-bg); ... }`.
  The `inline` modifier keeps the `var()` reference in generated CSS so :root stays the
  runtime authority — enabling reduced-motion overrides and future theming. NO
  `tailwind.config.ts`.
- **Reduced-motion via tokens**: `@media (prefers-reduced-motion: reduce) :root { --duration-*: 0ms }`
  zeroes every motion duration at the root, so any primitive consuming `--duration-*`
  (CSS keyframes or Framer Motion via CSS vars) auto-respects the preference without
  per-component logic.
- **Swiss discipline (ADR-0002)**: NO `--radius` or `--shadow` tokens — square corners,
  hairline rules only. Single accent color (#E63946 Carpathian red); B/W otherwise.
- **next/font + Tailwind v4 namespacing**: next/font's `variable: '--foo'` option sets
  `--foo` on `<html>` via the className applied in `layout.tsx`. To expose it as a
  design-system token name AND avoid an `@theme inline` self-reference cycle, namespace
  the next/font slot (`--font-inter`, `--font-jetbrains-mono`) and alias to the public
  token name (`--font-display`, `--font-mono`) at `:root`. Self-referential aliases like
  `@theme inline { --font-display: var(--font-display) }` would resolve to the initial
  value because :root and the next/font className live at the same element scope.
- **Inter "Display" via opsz**: SPEC says "Inter Display"; Google Fonts only exposes
  `Inter` as a single variable family. Pass `axes: ['opsz']` to `Inter()` from
  `next/font/google` and set `font-optical-sizing: auto` on body — the browser
  auto-picks the Display optical-size variant for large headings. One variable file
  ships, covering the full 100..900 weight axis. Leaner than per-weight static files.

**Learnings:**
- Tailwind v4's `--animate-*` token in `@theme inline` is the canonical way
  to register a custom animation utility. `--animate-marquee: marquee var(...)
  linear infinite` generates `.animate-marquee { animation: ... }` at build

  the @keyframes identifier.
- Marquee defense in depth is two-layer because the JS layer (useReducedMotion
  removing the class) is the user-facing path, but the CSS layer
  (--duration-marquee → 0s in @media) catches edge cases: hydration mismatch
  (during the first paint, useReducedMotion's getServerSnapshot returns false,
  so the class IS applied — but if duration is 0s under @media, no visible
  motion), or any future consumer who renders without the JS hook.

  React strips the attribute when `undefined` (so the first copy is left
  un-decorated for AT) and serializes "true" otherwise. Avoid the trap of
  `aria-hidden={i !== 0}` which would emit `aria-hidden="false"` on the
  first copy — valid HTML but extra noise that some AT engines parse oddly.
- Test mock for matchMedia in component-level tests differs from the hook
  test pattern: components don't need the listener-firing escape hatch
  (`__fire`) — they just need a stable initial `matches` value, since
  useReducedMotion reads via getSnapshot. Keeping the mock minimal
  (matches + addEventListener/removeEventListener as no-op spies) makes
  the test easier to read.
- `inline-flex w-max gap-[var(--space-12)]` is the trick for an inline
  marquee track that's wider than its parent: `w-max` makes the track
  take exactly the width of its inline-flex content, and `gap` separates
  the copies. Combined with `whitespace-nowrap` on the outer wrapper, no
  matter how long children is, the track lays out on one line and
  translateX(-50%) brings copy 2 to copy 1's start position.

**Learnings:**
- Compound-component state with React Context is straightforward in
  React 19. The two-context pattern (one for the Accordion's set
  state, one for the per-item identity + ARIA ids) keeps
  responsibilities clean and lets `AccordionTrigger` AND
  `AccordionContent` independently consume the item context without

  DOM ids when consumers can pass arbitrary `itemId` values (state
  keys may collide across separate Accordion instances; DOM ids must
  not). State scoping comes from context; DOM-id uniqueness comes from
  `useId()`. Keep the two concerns separate.
- DOM-walk-based keyboard nav (`querySelectorAll('[data-accordion-trigger]')`
  inside the keydown handler) is much simpler than the ref-array
  registration patterns Radix uses. The DOM is the source of truth for
  trigger order; React doesn't need to mirror it. Trade-off: pays a
  querySelectorAll on every relevant keystroke, which is negligible
  for typical accordion sizes.
- jsdom does NOT auto-fire `click` on `<button>` Enter/Space keydown
  the way real browsers do. Don't fight this — assert that triggers
  are `<button>` elements (so they get the behavior natively in
  real browsers) and skip simulating the synthetic click. Testing
  the platform contract (button semantics) is more durable than
  simulating jsdom workarounds.
- TS `noUncheckedIndexedAccess: true` in this repo means
  `const [a, b] = arr` types `a`/`b` as `T | undefined`. `fireEvent.*`
  parameters reject undefined. Wrap in a small `getTriggers(): [HE, HE, HE]`
  helper that asserts length and uses non-null assertions internally
  for a clean, typed tuple at every call site.
- Spreading a computed-key object as `{...{ [TRIGGER_ATTR]: "" }}` is
  a clean way to pass a `data-*` attribute whose name is held in a

**Learnings:**
- The mount-gate `useState(false) + useEffect(() => setMounted(true))`
  pattern is now blocked by eslint-plugin-react-hooks v6's
  `react-hooks/set-state-in-effect` rule. For SSR-safe portals,
  `typeof document === "undefined"` is sufficient if consumers
  initialize `open=false` (the realistic case) — server returns null,
  client first render also returns null (open=false), no hydration
  mismatch. Only edge case is `<Sheet open={true}>` on initial render,
  which would produce a brief mismatch warning; rare enough to ignore.
- React 19 + Next 16 'use client' components SSR by default. Any
  reference to `document` or `window` at the top level of render must
  be guarded. createPortal call lives behind `typeof document` check.
- Inert-restore pattern: capture prior state per element, NOT a single
  boolean. If a sibling already had `inert` (e.g., another modal in
  flight), we must NOT remove it on cleanup. `previousInert =
  siblings.map(s => s.hasAttribute("inert"))` then on cleanup
  `if (!previousInert[i]) s.removeAttribute("inert")` is the correct
  idempotent pattern.
- Focus-trap selector: `[tabindex]:not([tabindex="-1"])` only matches
  elements with an explicit tabindex attribute. Native focusables
  (button, a[href], inputs) need their own selectors because they're
  focusable WITHOUT an explicit tabindex. The 6-clause selector list
  covers the common ground; for production, an `aria-hidden=true`
  exclusion clause is a useful future addition.
- `useEffect` cleanup phase is the elegant focus-restore site: it
  fires on the dependency change BEFORE the new effect, and on unmount.
  Pattern: capture in effect body, restore in returned cleanup. Works
  symmetrically with the open: true → false transition.
- Framer Motion's `motion.div` forwards arbitrary `data-*` attributes
  to the DOM. Using `data-reduced-motion={reduced ? "true" : "false"}`
  on the panel gave the test a clean, framer-internals-free assertion
  point for "respects useReducedMotion" without depending on whether
  framer's animation actually runs in jsdom.
- Skipping AnimatePresence (no exit animation) avoids jsdom + framer
  flakiness around exit-onExitComplete timing. The trade-off: no
  visible exit transition. Acceptable for initial Sheet; AnimatePresence
  can be layered later without breaking API.
- `<button data-testid="x">x</button>` rendered inside a Sheet portal
  is correctly findable via `screen.getByTestId("x")` because
  testing-library queries `document.body` by default — and the portal
  mounts to `document.body`. No `within(panel)` indirection needed.
- For the inert-test, testing-library's `render` creates a fresh `<div>`
  appended to document.body and returns its container. With the Sheet
  portal also mounting to body, body has [tl-container, sheet-root]
  during open. Filter `body.children` by `!hasAttribute("data-sheet-root")`
  to assert all siblings (i.e., tl-container) get inert.
