# Ralph Progress Log

## Codebase Patterns

- **Tailwind v4 CSS-first (ADR-0005)**: Tokens live in `src/app/globals.css`.
  Pattern is `:root { --color-bg: #FAFAF7; ... }` + `@theme inline { --color-bg: var(--color-bg); ... }`.
  The `inline` modifier keeps the `var()` reference in generated CSS so :root stays the
  runtime authority — enabling reduced-motion overrides and future theming. NO
  `tailwind.config.ts`.
- **Reduced-motion via tokens**: `@media (prefers-reduced-motion: reduce) :root { --duration-*: 0ms }`
  zeroes every motion duration at the root, so any primitive consuming `--duration-*`
  (CSS keyframes or Framer Motion via CSS vars) auto-respects the preference without
  per-component logic.
- **Swiss discipline (ADR-0002)**: NO `--radius` or `--shadow` tokens — square corners,
  hairline rules only. Single accent color (#E63946 Carpathian red); B/W otherwise.
- **next/font + Tailwind v4 namespacing**: next/font's `variable: '--foo'` option sets
  `--foo` on `<html>` via the className applied in `layout.tsx`. To expose it as a
  design-system token name AND avoid an `@theme inline` self-reference cycle, namespace
  the next/font slot (`--font-inter`, `--font-jetbrains-mono`) and alias to the public
  token name (`--font-display`, `--font-mono`) at `:root`. Self-referential aliases like
  `@theme inline { --font-display: var(--font-display) }` would resolve to the initial
  value because :root and the next/font className live at the same element scope.
- **Inter "Display" via opsz**: SPEC says "Inter Display"; Google Fonts only exposes
  `Inter` as a single variable family. Pass `axes: ['opsz']` to `Inter()` from
  `next/font/google` and set `font-optical-sizing: auto` on body — the browser
  auto-picks the Display optical-size variant for large headings. One variable file
  ships, covering the full 100..900 weight axis. Leaner than per-weight static files.

### Iteration 3 — US-003: cn() utility + 3 hooks (TDD)
**Date:** 2026-05-09T18:48Z
**Quality Tier:** 1
**Health Check:** PASS (2 tests passed pre-work, 0 type/lint errors)
**Implemented (strict TDD — RED → GREEN for each):**
- `src/lib/cn.ts` — recursive class-merger: flattens nested arrays, drops
  falsy (false/null/undefined/0/""), splits on whitespace so `cn("a  b","c")`
  → `"a b c"`. Stringifies non-zero numbers (so `count > 0 && count` patterns
  still emit a class). Type `ClassValue = string | number | false | null
  | undefined | ClassValue[]`.
- `src/hooks/useReducedMotion.ts` — `useSyncExternalStore` over
  `window.matchMedia('(prefers-reduced-motion: reduce)')`. Subscribes to
  the change event; getServerSnapshot returns false (SSR-safe); React
  drives consistency, no manual setState in effect (lint clean).
- `src/hooks/useEscapeKey.ts` — single `keydown` listener on `window`,
  filters to `event.key === "Escape"` only. Latest-callback pattern via
  `useRef` so consumers can pass inline functions without re-binding the
  listener every render. Cleanup on unmount.
- `src/hooks/useScrollProgress.ts` — IntersectionObserver with 101
  thresholds (0.00..1.00 step 0.01). Computes monotone progress
  `clamp01((rootBounds.height - rect.top) / (rootBounds.height + rect.height))`
  → 0 as element enters from bottom, 1 as it exits to top. Falls back to
  `intersectionRatio` when `rootBounds` is null (cross-origin iframe).
  NO scroll handler — verified by test that asserts
  `window.addEventListener('scroll', ...)` was never called.

**Learnings:**
- `useSyncExternalStore` is the canonical fix for `react-hooks/set-state-in-effect`
  when the value comes from a subscribable browser store like matchMedia.
  Pattern: pure `subscribe` + pure `getSnapshot` + `getServerSnapshot` for
  SSR. React handles the consistency tear and snapshot stability — no
  setState inside useEffect needed.
- IntersectionObserver gives "scroll progress through viewport" without
  any scroll handler. Use 101 thresholds (`Array.from({length:101}, (_,i)=>i/100)`)
  so callbacks fire on every ~1% intersection step, then compute monotone
  progress from `boundingClientRect.top + rootBounds.height`. Tested by
  spying `window.addEventListener` and asserting NO `'scroll'` calls.
- `react-hooks/set-state-in-effect` (eslint-plugin-react-hooks v6) is strict
  about ANY direct setState in an effect body — even guarded ones. Move to
  external-store hook or callback ref pattern.
- v8 coverage `include` from vitest.config.ts (`src/lib/**`, `src/hooks/**`)
  emits ONLY those paths, but the `coverage/` HTML output triggers eslint
  scans on generated files. /coverage is gitignored; the 2 warnings are
  cosmetic. Could add `coverage/` to eslint `ignores` in a follow-up.
- `Mock<(args) => void>` type from vitest gives strict callable types
  for vi.fn() return values — better than `ReturnType<typeof vi.fn>`
  which is `Mock<Procedure | Constructable>` and trips TS2348 on
  function-call usage.
- Latest-callback ref pattern: store callback in `useRef`, sync it in a
  separate `useEffect([cb])`, and call `ref.current()` from the listener.
  Lets consumers pass inline functions without churning the listener.
